Error

class Error : public Exception

Signals that a XML-RPC error has occurred during the XML-RPC request. This exception may be thrown by top (application) layer of XML-RPC stack, for example on invalid cast of XML-RPC values.

Fault

class com::kassowrobots::api::robot::xmlrpc::Fault : public Exception

Signals that a XML-RPC fault has occurred during the XML-RPC request. This exception may be thrown by lower (communication & processing) layers of XML-RPC stack, for example if requested XML-RPC method does not exist.

Public Functions

inline int getCode()

Retrieves the fault code.

Returns

One of the following fault code constants:

  • CODE_PARSE_ERROR_NOT_WELL_FORMED

  • CODE_INVALID_XML_RPC

  • CODE_METHOD_NOT_FOUND

  • CODE_INVALID_METHOD_PARAMETERS

  • CODE_SYSTEM_ERROR

  • CODE_TRANSPORT_ERROR

inline String getDescription()

Retrieves the fault description.

Returns

User friendly description of this fault.

Public Static Attributes

static final int CODE_PARSE_ERROR_NOT_WELL_FORMED   = -32700

Code constant returned by getCode(): Signals that the request or response XML cannot be parsed, because the XML is not well formed.

static final int CODE_INVALID_XML_RPC   = -32600

Code constant returned by getCode(): Signals that the request or response XML can be parsed, but the content does not conform to XML-RPC specification.

static final int CODE_METHOD_NOT_FOUND   = -32601

Code constant returned by getCode(): Signals that the requested XML-RPC method was not found, ie. that the method with the given name was not registered to the XML-RPC server.

static final int CODE_INVALID_METHOD_PARAMETERS   = -32602

Code constant returned by getCode(): Signals that the requested XML-RPC method params do not match the registered method params on the XML-RPC server side.

static final int CODE_SYSTEM_ERROR   = -32400

Code constant returned by getCode(): Signals that the XML-RPC request failed with some unexpected internal error.

static final int CODE_TRANSPORT_ERROR   = -32300

Code constant returned by getCode(): Signals that the XML-RPC request failed due to the communication error between the teach pendant and the cabinet.

Signals a variable that is registered in program var register and its value is shared among the sequences.

KRContext

class com::kassowrobots::api::app::KRContext : public ContextWrapper

Interface to global information about CBun app environment. It allows access to CBun app specific resources (such as strings, drawables, etc.) and classes as well as KR services.

Public Functions

inline Object getKRService(String aName)

Returns the handle to a KR-level service by name. The class of the returned object varies by the requested name. Currently available names are:

TEACHING_MANAGER_SERVICE ("teaching_manager")

A TeachingManager for subscribing teaching events which let you to override behaviour of default teaching handler.

SELECTION_MANAGER_SERVICE ("selection_manager")

A SelectionManager for controlling and monitoring selection of variables.

DRAG_MANAGER_SERVICE ("drag_manager")

A DragManager for subscribing drag events by views.

Note: KR services obtained via this API are available (non null) only if the CBun app is launched in the host (Boson) application.

XmlRpcClient

class com::kassowrobots::api::robot::xmlrpc::XmlRpcClient

Provides an interface for execution of XML-RPC requests. This XML-RPC client is designed to be used with an XML-RPC server in 1-N relation, ie. one client can communicate with one server, but there might be multiple clients for the single server. All RPC methods needs to be registered to the XML-RPC server prior their execution, otherwise the XML-RPC request will fail.

Public Functions

inline XmlRpcClient(String aServerId)

Constructs the XML-RPC client for the given server. The server is identified by the server ID, which is defined as a backend ‘group’ attribute in bundle.xml, ie. the server ID has to be aligned with the unique name of CBun Sandbox process.

Parameters

aServerId – Unique identifier of the server.

inline Value execute(String aMethodName, Params aParams, double aTimeout)

Executes the XML-RPC request. This method sends an XML-RPC request to the XML-RPC server, the server executes the requested method and sends the response back to the client. The XML-RPC method is identified by its name and it has to be registered to the XML-RPC server prior its execution. Whereas multiple arguments can be passed to this method via params, there is just single return value. If the XML-RPC method is not executed withing the timeout interval, an exception is thrown.

Parameters

  • aMethodName – Name of the RPC method to be executed.

  • aParams – XML-RPC params to be passed to the RPC method.

  • aTimeout – XML-RPC timeout in seconds.

Throws

Fault – If XML-RPC request fails.

Returns

Return value of XML-RPC request.
